TEC — GTFS

GET /tec/gtfs

Download the official TEC static schedule as a GTFS .zip archive. This dataset provides full transit schedules for Wallonia and is suitable for mapping, analytics, and real-time overlays.

Description

TEC (Wallonia) publishes a GTFS static feed containing route, trip, stop, calendar, and shape definitions. This endpoint serves the GTFS ZIP archive as provided by TEC and indexed historically by the MobilityTwin.Brussels platform. The feed can be used for validation, visualization, or integration with GTFS-RT data.

Query Parameters

NameTypeRequiredDescription
timestamp integer No If omitted, returns the latest available GTFS snapshot. If provided, returns the snapshot closest to or before the given timestamp (≤ logic).

Output Format

Content-Type: application/zip

The response body is a binary ZIP archive containing standard GTFS .txt files (CSV text format).

Contents of the GTFS Zip (typical)

  • agency.txt
  • routes.txt
  • trips.txt
  • stop_times.txt
  • stops.txt
  • calendar.txt, calendar_dates.txt
  • shapes.txt
  • Optionally: frequencies.txt, transfers.txt, and other GTFS-compliant files

Estimated Response Volume

  • Typical ZIP size: tens of MB (varies by update and route coverage).
  • Snapshots are versioned and retrieved according to publication cadence.

Data Source & Transformation Pipeline

Data Source

Provider: TEC (Wallonia) — official GTFS static feed.

Pipeline

  1. Ingestion: MobilityTwin periodically fetches the official TEC GTFS ZIP feed.
  2. Versioning: Each feed is stored and indexed by timestamp for historical queries.
  3. Serving: The ZIP file is served as-is with ≤ timestamp selection logic.

Example Application Scenarios

  • Generate maps of bus routes and stops across Wallonia.
  • Build schedule and timetable tools for passengers.
  • Integrate GTFS static data with GTFS-RT for real-time applications.
  • Perform accessibility, coverage, or frequency analysis.

Operational Context for Wallonia Regional Transit Ecosystem

The TEC GTFS endpoint provides complete scheduled bus service data for Wallonia (Belgium's French-speaking southern region)—covering cities like Liège, Namur, Charleroi, Mons, and rural communities across provinces of Hainaut, Namur, Liège, Luxembourg, and Brabant Wallon. For regional mobility planners, TEC GTFS enables comprehensive coverage analysis: identifying underserved rural areas in Ardennes valleys, evaluating commuter corridor frequencies (Charleroi-Brussels, Liège-Verviers), and quantifying inter-city connectivity complementing SNCB rail service. The GTFS format standardization allows multi-modal journey planners to seamlessly integrate TEC buses with SNCB trains and STIB Brussels transit—passenger traveling from rural Walloon village to Brussels combines TEC local bus to regional hub, SNCB train to Brussels, and STIB metro to final destination, all planned through unified GTFS-based routing. Accessibility researchers use stops.txt coordinates and calendar patterns to evaluate rural mobility equity: do elderly residents in Ardennes municipalities have daily bus access to healthcare/shopping, or only limited school-day service? TEC's regional scope means routes balance urban frequency (Liège city buses every 10-20 minutes) with rural coverage (small villages served twice daily), reflected in calendar.txt patterns and frequencies.txt headways. For developers, historical GTFS snapshots enable evolution tracking: has TEC service frequency increased in response to car-free initiatives, have new routes appeared connecting SNCB rail stations for improved intermodal transfer, do seasonal patterns exist (enhanced service to Ardennes ski resorts winter, reduced service to universities during summer)? The French-language context (contrast with Dutch-speaking Flanders' De Lijn operator) appears in stop_name fields and agency metadata—critical for user interface localization serving Wallonia's predominantly French-speaking population while supporting multilingual Brussels connections.
