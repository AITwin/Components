Telraam

GET /traffic/telraam

Provides near real-time multimodal traffic observations for Telraam street segments in Brussels, returned as GeoJSON.

Description

Returns live traffic counts from Telraam sensors across Brussels street segments. The data includes mode-specific counts (cars, bikes, pedestrians, heavy vehicles), uptime metrics, reporting intervals, and speed indicators such as v85. Each segment is enriched with geometry for direct use in mapping and analysis tools. If a timestamp is not specified, the latest snapshot is returned; otherwise, the closest ≤ snapshot is used.

Output Format

Content-Type: application/json (GeoJSON)

Returns a GeoJSON FeatureCollection where each feature represents a monitored street segment and its current metrics.

Data Source & Transformation Pipeline

Data Source

  • Telraam Live API — near real-time multimodal traffic data.

Pipeline

  1. Authentication: MobilityTwin retrieves snapshots via Telraam’s live API using secure keys.
  2. Ingestion: Fetch and harmonize data for each Telraam segment.
  3. GeoJSON Assembly: Attach geometry and normalize fields.
  4. Serving: Return the latest or nearest ≤ snapshot as a GeoJSON FeatureCollection.

Operational Context: This Telraam traffic monitoring feed represents a transformative approach to urban traffic intelligence—citizen-powered sensors providing modal disaggregation (car/bike/pedestrian/heavy) and speed data at granular street-segment level, filling massive gaps in traditional infrastructure-based monitoring that focuses on major corridors while leaving residential streets unmeasured. For city traffic planners and mobility managers, Telraam data enables evidence-based interventions: identifying residential streets with inappropriate through-traffic (high car volumes, speeding evident in v85 >40 km/h in 30 zones), demonstrating latent cycling demand on corridors lacking infrastructure (justifying bike lane investments), and measuring before/after impacts of traffic calming (quantifying volume reductions, speed decreases). For neighborhood advocates and community groups, the data democratizes traffic intelligence—residents can monitor their own streets, document speeding or truck intrusions with objective counts, and build data-driven cases for traffic management changes (speed cameras, turn restrictions, modal filters). For researchers and policy analysts, Brussels-wide Telraam coverage enables unprecedented mobility pattern studies: mode share calculations at neighborhood scale (tracking progress toward sustainable mobility targets), temporal analysis of car dependency (weekday vs. weekend patterns), weather/seasonal impacts on cycling volumes, and equity analyses (comparing traffic impacts across socioeconomic neighborhoods). For app developers and civic tech builders, the feed supports real-time neighborhood traffic dashboards, automated speeding alerts for community groups, before/after intervention visualizations, and integration with other mobility data (correlating car volumes with air quality, transit ridership, micromobility usage). The Telraam citizen science model—volunteers hosting window-mounted sensors, contributing to collective traffic intelligence—creates dense monitoring networks impossible with traditional infrastructure approaches, while the modal disaggregation (distinguishing cars/bikes/pedestrians/heavy at individual-vehicle level) provides insights critical for 21st-century sustainable mobility planning that car-only counts cannot deliver.
